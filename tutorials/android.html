Kiosk Mode (Without Root)

Since Android doesn't let you have full control over the whole operating system, you are quite limited as to what you can do to fully "lock-down" your device with your application (i.e. Kiosk mode). However, there are some workarounds to this if you spend enough time to tinker around. To simulate kiosk mode on Android for a full lockdown on an Android application, there are a few things you need to control. NOTE: THIS REQUIRES FULL ACCESS TO THE DEVICE YOU ARE APPLYING THE KIOSK MODE ON

  1. Status bar
  2. Back button
  3. Home button
  4. Multitask button
  5. Power and volume button

1. Status Bar

The easiest way to hide the status bar dropdown is to either hide it completely by setting the application to full screen


However, if you do not want the user to fully deny users access to the status bar (to navigate out of the application) there is a simply hacky way of achieving this by creating an overlay that would rest on top of your application that will consume all touch inputs from the user (i.e. swiping down the status bar).

This code essentially draws a transparent bar on top of the status bar which consumes all touch events.

2. Back Button

It's as simple as manually overriding the back button (Note: This disables back button navigation within your app. If you do not want to do this, simply override this in your activity and you'll be able to navigate back again)

3. Home Button

This is a bit tricky to completely override. For this to actually be overriden, you need to set your application as the main launcher of the device itself. This can be easily done in the settings of the device and choosing your application as the default launcher for the device.

4. Multitask Button

The multitasking button allows you to switch between running apps and close them. This is not desireable! To have this button on lockdown, we need to override the method that is called whenever your app is toggled out. The onPause method is responsible for this. Basically, whenever you toggle out of your app, the onPause method is called.

This code basically forces your application to be at the front of all the applications open all the time which renders the multitasking button useless.

5. Power and Volume Buttons

Disabling the power and volume buttons are also quite tricky since they are received from the OS side. However, you can intercept these button presses and execute your own code by creating a receiver that will listen for the button presses.

Don't forget to register this receiver in the start of your main activity so that it continuously listens in the background for these button presses.

Saving to Shared Preferences

When you need to save something temporarily (i.e. tokens, passwords, settings, etc...) Android allows you to save them to shared preferences. Shared preferences is a file specific to your application to which only your application can access. With this, you can read and write to this whatever you want. This is very useful when you want to save a setting for example that you want to be unique for every device (Ex. language).
